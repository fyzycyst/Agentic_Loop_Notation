# Contributing to ALN

Thank you for helping make Agentic Loop Notation clearer and more useful.

ALN v0.1 is intentionally small. Contributions should make loops easier to read, audit, teach, or reuse without turning the notation into a heavy runtime or policy system.

## Example Submissions

Good examples should include:

- a concrete loop target
- a clear goal
- required context
- explicit authority boundaries
- one fixed verifier
- bounded actions
- keep, revert, and escalate rules
- distinct success, no-op, stall, budget, and blocked stop states
- return artifacts a human can inspect

If an example is adapted from an existing loop, prompt, article, video, repo, or workflow, include source attribution and a link when possible.

## Review Checklist

Before opening an issue or pull request, check:

- Does the loop return evidence rather than only claims?
- Is the verifier repeatable?
- Could the loop accidentally imply unsafe permission?
- Are external sources credited?
- Is the contribution small enough for v0.1?

## Scope

In scope:

- ALN notation clarity
- examples
- Loop Doctor review guidance
- Codex mapping and other runtime mappings
- attribution and citation improvements

Out of scope for v0.1:

- a parser or compiler
- a hosted registry
- a formal governance process
- claims that ALN provides authorization or safety by itself
